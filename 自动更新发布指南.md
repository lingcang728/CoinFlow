# CoinFlow 自动更新发布指南

本应用已接入自动更新（`electron-updater`）。家人在「关于 → 检查更新」点一下即可自动下载并安装新版本，**不需要先卸载旧版，数据也不会丢失**。

更新文件托管在**国内对象存储**（阿里云 OSS / 腾讯云 COS / 七牛云），大陆下载快、稳定、无需梯子。

---

## 一、一次性配置（只做一次）

1. **开通一个对象存储 Bucket**（任选其一：阿里云 OSS / 腾讯云 COS / 七牛云）：
   - 新建一个 Bucket，**读权限设为「公共读」**（这样程序能匿名下载更新文件）。
   - 获取它的**访问域名**（如 `https://coinflow-xxxx.oss-cn-hangzhou.aliyuncs.com` 或你绑定的 CDN 自定义域名）。
   - 在 Bucket 里建一个目录，例如 `coinflow/`。

2. **把下载地址填进 `package.json`**：
   找到 `build.publish` 段，把占位域名换成你的真实地址（**末尾要带斜杠**）：
   ```json
   "publish": [
     {
       "provider": "generic",
       "url": "https://你的域名/coinflow/",
       "channel": "latest"
     }
   ]
   ```

3. **重新打包一次**，让新地址写进安装包：
   ```powershell
   npm run build:desktop
   ```

---

## 二、每次发新版本的流程

1. **递增版本号**（`package.json` 与 `package-lock.json`，例如 `1.0.5` → `1.0.6`）。
   > ⚠️ 版本号只能往上加，不能回退或复用，否则自动更新判断不出新版本。

2. **打包**：
   ```powershell
   npm run build:desktop
   ```
   产物在 `release/` 下，需要上传的就这三个文件：
   - `latest.yml`（更新清单，**必须覆盖上传**）
   - `CoinFlow-Setup-<版本>.exe`（安装包）
   - `CoinFlow-Setup-<版本>.exe.blockmap`（增量更新差分，让家人只下载变化的部分）

3. **上传到对象存储**：把上面三个文件传到 Bucket 的 `coinflow/` 目录（`latest.yml` 覆盖旧的）。

4. **完成**。家人打开应用 →「关于 → 检查更新」→ 自动下载 → 点「立即重启并安装」即可。

---

## 三、家人首次安装

- 自动更新的前提是家人先装过一次。**第一次**把 `CoinFlow-Setup-<版本>.exe` 用任意方式发给他们（微信、U 盘、网盘都行），双击安装即可。
- 之后的所有版本，他们都能在应用内一键更新，不用你再发安装包。

---

## 四、注意事项

- **首次安装的安全提示**：安装包未做代码签名，Windows 可能弹「Windows 已保护你的电脑 / 未知发布者」。让家人点 **「更多信息」→「仍要运行」** 即可。
  （如果想去掉这个提示，需要购买代码签名证书，可选、非必需。）
- **安装位置**：一键安装到当前用户目录 `%LOCALAPPDATA%\Programs\CoinFlow`，不需要管理员、不会乱装到别的盘。
- **数据位置**：账本保存在 `%APPDATA%\CoinFlow`，**更新永不丢失**。
- **卸载**：Windows「设置 → 应用 → CoinFlow → 卸载」，或开始菜单里的卸载器，一键卸载。
- **检查更新失败**：通常是 `publish.url` 没填对、对象存储不是公共读、或三个文件没传全。确认 `latest.yml` 能在浏览器里直接打开（`https://你的域名/coinflow/latest.yml`）。
